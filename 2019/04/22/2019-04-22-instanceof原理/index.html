问题：typeof能干什么？
回答：判断基本类型的，比如：Number, Boolean, Undefined, String, Null， 其他的引用类型和Null会返回object；

问题：如果一个对象是数组，应该怎么判断？
回答：用instanceof，比如对象是arr，用arr instanceof Array，如果返回ture就表示是数组，为false就不是。

instanceof原理

什么是instanceof？

instanceof运算符用于测试构造函数的prototype属性是否出现在对象的原型链中的任何位置

五、结论

1、instanceof 的作用

用于判断一个引用类型是否属于某构造函数；

还可以在继承关系中用来判断一个实例是否属于它的父类型。

instanceof运算符用于测试构造函数的prototype属性是否出现在对象的原型链中的任何位置

继承关系判断父类型

非继承关系是：引用类型是否属于构造函数。

2、和typeof的区别：

typeof在对值类型number、string、boolean 、null 、 undefined、 以及引用类型的function的反应是精准的；但是，对于对象{ } 、数组[ ] 、null 都会返回object

为了弥补这一点，instanceof 从原型的角度，来判断某引用属于哪个构造函数，从而判定它的数据类型。
